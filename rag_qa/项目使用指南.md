## 快速开始

### 环境准备

```bash
# 安装依赖
pip install -r requirements.txt

# 确保 Milvus 服务运行
docker run -d --name milvus-standalone \
  -p 19530:19530 \
  -p 9091:9091 \
  milvusdb/milvus:latest
```

### 配置文件

编辑 `config.ini`，设置必要的参数：

```ini
[milvus]
host = localhost
port = 19530
database_name = itcast
collection_name = edurag_final

[llm]
model = qwen-plus
dashscope_api_key = sk-your-api-key
dashscope_base_url = https://api.yunwu.ai/v1

[retrieval]
parent_chunk_size = 1200
child_chunk_size = 300
chunk_overlap = 50

candidate_m = 2
```

## 核心功能

### 1. 查询分类

系统自动识别查询类型：

```python
from core.query_classifier import QueryClassifier

classifier = QueryClassifier(model_path="bert_query_classifier_new")

# 预测查询类型
category = classifier.predict_category("露天矿山开采工艺流程是什么？")
print(category)  # 输出："专业咨询" 或 "通用知识"
```

**支持两类**:
- **通用知识**: 直接调用 LLM 回答（如"什么是 AI?"）
- **专业咨询**: 执行 RAG 检索（如"充填采矿法的适用条件"）

### 2. 智能检索策略

自动选择最优检索策略：

```python
from core.strategy_selector import StrategySelector

selector = StrategySelector()
strategy = selector.select_strategy("对比露天开采和井下开采的优缺点")
print(strategy)  # 输出："子查询检索"
```

**四种策略**:
- **直接检索**: 简单明确的问题
- **假设问题检索 (HyDE)**: 抽象概念问题
- **子查询检索**: 对比/多因素问题
- **场景重构检索**: 复杂工程问题

### 3. 混合检索与重排序

```python
from core.vector_store import VectorStore

vector_store = VectorStore()

# 执行混合检索（稠密 + 稀疏向量）+ BGE-Reranker 重排序
results = vector_store.hybrid_search_with_rerank(
    query="深部开采地压控制",
    k=5,
    source_filter="mining"
)

for doc in results:
    print(f"来源：{doc.metadata['source']}")
    print(f"内容：{doc.page_content[:100]}...")
```

### 4. 多轮对话管理

```python
from core.new_rag_system import RAGSystem

rag_system = RAGSystem(vector_store, call_llm)

# 第一轮
answers = list(rag_system.generate_answer(query="什么是机器学习？", use_history=True))

# 第二轮（自动带上前一轮历史）
answers = list(rag_system.generate_answer(query="它有哪些算法？", use_history=True))

# 加载旧会话
answers = list(rag_system.generate_answer(
    query="继续刚才的话题",
    session_id="session_20260402_xxxxx",
    use_history=True
))
```

### 5. 知识库溯源

```python
from view_sources import SourceViewer

viewer = SourceViewer()

# 查看检索来源
viewer.view(query="露天开采工艺", top_k=5)

# 导出为 JSON/CSV/Markdown
viewer.export_to_json("sources.json")
viewer.export_to_csv("sources.csv")
viewer.export_to_markdown("report.md")
```

## 使用示例

### 场景 1: 数据处理（构建知识库）

```bash
# 处理 data 目录下的所有文档并入库
python rag_main.py --data-processing --data-dir ./data
```

**支持的文档格式**:
- PDF (.pdf) - 自动 OCR
- Word (.docx) - 自动 OCR
- PPT (.ppt/.pptx) - 自动 OCR
- 图片 (.jpg/.png) - 自动 OCR
- 文本 (.txt/.md)

**处理流程**:
1. 加载文档 → 2. OCR 识别 → 3. 分层切分（父块 1200 tokens → 子块 300 tokens）
4. 生成向量（BGE-M3） → 5. 批量插入 Milvus（每批 500 个）

### 场景 2: 交互式问答

```bash
# 启动查询模式
python rag_main.py
```

**交互示例**:

### 场景 3: 查看检索来源

```bash
# 交互模式查看溯源

# 命令行查询
python view_sources.py -q "充填采矿法适用条件" -k 5

# 导出溯源报告
python view_sources.py -q "深部开采技术" --export md -o report.md
```

### 场景 4: RAGAS 评估

```bash
# 准备评估数据 (rag_evaluate_data.json)
# 格式: [{"question": "...", "answer": "...", "context": [...], "ground_truth": "..."}]

# 执行评估
cd rag_assesment
python rag_as.py
```

**评估指标**:
- Faithfulness (忠实度): 答案是否基于上下文
- Answer Relevancy (相关性): 答案与问题匹配度
- Context Precision (精确度): 上下文相关性
- Context Recall (召回率): 上下文完整性

### 场景 5: 训练自定义分类器

```bash

python generate_training_data.py

# 2. 训练 BERT 分类器
python train_classifier.py

python test_query.py
```

## 常见问题

### Q1: Milvus 连接失败

**错误**: `Failed to connect to Milvus`

**解决**:
```bash
# 检查 Milvus 是否运行
docker ps | grep milvus

# 重启 Milvus
docker restart milvus-standalone

# 检查防火墙
telnet localhost 19530
```

### Q2: CUDA 不可用

**错误**: `Found no NVIDIA driver on your system`

**解决**:
```python
# 修改 core/vector_store.py
self.device = 'cuda' if torch.cuda.is_available() else 'cpu'

# 或设置环境变量
os.environ["CUDA_VISIBLE_DEVICES"] = ""  # 强制使用 CPU
```

### Q3: OCR 处理速度慢

**优化**:
```python
# 修改 edu_document_loaders/edu_pdfloader.py
PDF_OCR_THRESHOLD = (0.3, 0.3)  # 降低阈值，减少 OCR 图片数量

# 启用缓存
loader = OCRPDFLoader(file_path, use_cache=True)
```

### Q4: 显存不足

**解决**:
```python
# 减小批次大小
batch_size = 8  # 原为 16

# 使用 FP16
use_fp16 = True  # GPU 时启用混合精度

# 清理缓存
import torch
torch.cuda.empty_cache()
```

### Q5: 如何备份知识库

```bash
# 导出 Milvus 数据
python -c "
from core.vector_store import VectorStore
import json

vs = VectorStore()
results = vs.client.query(
    collection_name='edurag_final',
    filter='',
    output_fields=['text', 'source', 'parent_id'],
    limit=10000
)

with open('knowledge_backup.json', 'w') as f:
    json.dump(results, f, ensure_ascii=False, indent=2)
"
```

### Q6: 如何添加新学科

```bash
# 1. 在 config.ini 中添加学科
valid_sources = ["mining", "ai", "java", "your_subject"]

# 2. 创建数据目录
mkdir data/your_subject_data

# 3. 放入文档（PDF/Word/PPT 等）

python rag_main.py --data-processing --data-dir ./data
```

### C. 性能优化建议

1. **批量处理**: 文档入库时使用 `batch_size=500`
2. **GPU 加速**: 启用 CUDA 和 FP16 混合精度
3. **索引优化**: IVF_FLAT 的 nlist=128 适合万级数据
4. **缓存策略**: OCR 结果缓存避免重复处理
5. **分层检索**: 子块检索 → 父块返回，平衡精度和语义
